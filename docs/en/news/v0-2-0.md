JekyllNet `v0.2.0` is now released. This version focuses on turning existing capabilities into a clearer, faster adoption path through stronger documentation structure and practical guidance.

## Official links

- Repository: [https://github.com/JekyllNet/JekyllNet](https://github.com/JekyllNet/JekyllNet)
- Website: [https://jekyllnet.help](https://jekyllnet.help)
- GitHub Pages entry: [https://jekyllnet.github.io/JekyllNet/](https://jekyllnet.github.io/JekyllNet/)

## Highlights in v0.2.0

- Added a complete end-to-end usage guide
- Reorganized docs navigation by category and intent
- Improved handoff between release news and practical guides
- Filled bilingual gaps to keep Chinese and English documentation aligned

## Who this release helps most

- Teams moving Jekyll-style sites into a .NET workflow
- Projects that want GitHub Pages style behavior with explicit CLI and CI control
- Documentation teams running multilingual content with AI-assisted translation
